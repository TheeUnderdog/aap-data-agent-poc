AGENT GROUP RATIONALE
---------------------
Five agents designed around the distinct business personas who consume loyalty 
program data at an auto parts retailer. Each agent owns a clear scope with 
defined view access, guardrails, and cross-agent referrals for out-of-scope 
questions. All agents query the same RewardsLoyaltyData semantic model.

Design principles:
  • One agent per business function — no overlap in primary responsibility
  • Every semantic view is covered by at least one agent as primary
  • Cross-agent referrals create a connected experience (no dead ends)
  • Guardrails are consistent: no PII in aggregates, no fabrication, no predictions
  • Examples use realistic auto parts retail numbers and terminology
