## Key Decisions

- **Fabric REST API with Azure CLI tokens** — Uses `api.fabric.microsoft.com/v1` with token authentication
- **Idempotent by design** — Scripts check for existing resources; safe for repeated execution
- **Dual token audiences** — Management API and SQL endpoint use different token audiences
- **`.env.fabric` as config bridge** — Setup script outputs; deploy script reads; other scripts consume
- **SqlClient fallback** — Deploy script doesn't require SqlServer PowerShell module
- **External SQL ownership** — Livingston owns `create-semantic-views.sql`; provisioning scripts delegate to it
