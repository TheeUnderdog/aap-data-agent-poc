## Technical Details

### Architecture
- **Frontend:** React SPA (static assets served by Container Apps ingress)
- **Backend:** Flask Python application with gunicorn WSGI server
- **Auth:** MSAL (Microsoft Authentication Library) middleware for Entra ID SSO
- **Data Access:** Managed identity with workspace Contributor role → Fabric API
- **Deployment:** GitHub Actions → ghcr.io → Azure Container Apps

### Configuration
- `.env` variables for client credentials and Fabric workspace ID
- Managed identity auto-discovery via Azure CLI/runtime
- MSAL redirect URI points to Container Apps FQDN
- CORS headers configured for SPA-backend communication

### Deployment Flow
1. GitHub Actions triggered on push to `master`
2. Build multi-stage Docker image (dependencies, Python app, static assets)
3. Push to `ghcr.io/aap-data-agent/web:latest`
4. PowerShell script or ACA portal deploys container, binds managed identity
5. Fabric API calls use managed identity (no secrets in container)
