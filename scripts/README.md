# Provisioning Scripts

Automated setup for the AAP Data Agent POC Fabric environment. **No portal clicking required.**

---

## Prerequisites

| Requirement | Version | Check |
|---|---|---|
| PowerShell | 7.0+ | `$PSVersionTable.PSVersion` |
| Azure CLI | 2.50+ | `az --version` |
| Azure subscription | With Fabric capacity | `az account show` |
| Fabric capacity | F64 or higher recommended | Get ID from Fabric admin portal |
| SqlServer module | *(optional, for view deployment)* | `Install-Module SqlServer` |

## Quick Start

```powershell
# 1. Login to Azure
az login

# 2. Provision workspace + Lakehouse
./scripts/setup-workspace.ps1 -CapacityId "YOUR-CAPACITY-GUID"

# 3. Import and run the data generation notebook in the Fabric workspace
#    (see docs/data-schema.md for notebook details)

# 4. Deploy semantic views
./scripts/deploy-semantic-views.ps1
```

## Scripts

### `setup-workspace.ps1`

Creates (or reuses) a Fabric workspace and Lakehouse.

**Parameters:**

| Parameter | Required | Default | Description |
|---|---|---|---|
| `-CapacityId` | ✅ | — | Fabric capacity GUID |
| `-WorkspaceName` | — | `AAP-RewardsLoyalty-POC` | Workspace display name |
| `-LakehouseName` | — | `RewardsLoyaltyData` | Lakehouse display name |
| `-WhatIf` | — | — | Preview changes without executing |

**What it produces:**
- Fabric workspace assigned to your capacity
- Lakehouse with SQL endpoint
- `scripts/.env.fabric` config file with IDs and connection info

**Examples:**

```powershell
# Default names, specify capacity
./scripts/setup-workspace.ps1 -CapacityId "a1b2c3d4-e5f6-7890-abcd-ef1234567890"

# Custom names
./scripts/setup-workspace.ps1 `
    -WorkspaceName "MyTeam-POC" `
    -LakehouseName "LoyaltyData" `
    -CapacityId "a1b2c3d4-e5f6-7890-abcd-ef1234567890"

# Dry run
./scripts/setup-workspace.ps1 -CapacityId "..." -WhatIf
```

### `deploy-semantic-views.ps1`

Deploys the 7 contract views to the Lakehouse SQL endpoint.

**Parameters:**

| Parameter | Required | Default | Description |
|---|---|---|---|
| `-SqlEndpoint` | — | from `.env.fabric` | Lakehouse SQL endpoint |
| `-SqlFile` | — | `scripts/create-semantic-views.sql` | Path to SQL file |
| `-DatabaseName` | — | from `.env.fabric` | Lakehouse database name |
| `-WhatIf` | — | — | Preview which views would be deployed |

**Prerequisites:**
- Run `setup-workspace.ps1` first (or provide `-SqlEndpoint` manually)
- Run the data generation notebook so underlying tables exist
- The `create-semantic-views.sql` file must exist (provided by Livingston)

**Examples:**

```powershell
# Use defaults from .env.fabric
./scripts/deploy-semantic-views.ps1

# Explicit endpoint
./scripts/deploy-semantic-views.ps1 -SqlEndpoint "xyz.datawarehouse.fabric.microsoft.com"
```

## Step-by-Step Walkthrough

### Phase A: Local/Dev Setup

1. **Login to Azure**
   ```powershell
   az login
   az account set --subscription "YOUR-SUBSCRIPTION"
   ```

2. **Find your Fabric capacity ID**
   - Go to [Fabric Admin Portal](https://app.fabric.microsoft.com/admin-portal/capacities) → Capacities
   - Copy the capacity GUID
   - Or use: `az rest --method get --url "https://api.fabric.microsoft.com/v1/capacities"` to list capacities

3. **Provision the workspace**
   ```powershell
   ./scripts/setup-workspace.ps1 -CapacityId "<your-capacity-guid>"
   ```
   This creates the workspace and Lakehouse, and saves IDs to `.env.fabric`.

4. **Import the data generation notebook**
   - Open the workspace in Fabric portal
   - Import the notebook from `fabric/notebooks/` (provided by Livingston)
   - Attach the notebook to the Lakehouse
   - Run all cells to generate placeholder data

5. **Deploy semantic views**
   ```powershell
   ./scripts/deploy-semantic-views.ps1
   ```
   This creates the 7 contract views that the Data Agent and backend will query.

6. **Verify**
   - In Fabric portal, open the Lakehouse SQL endpoint
   - Run: `SELECT * FROM v_member_summary LIMIT 10`
   - All 7 views should return data

### Phase B: Production Deployment

Same scripts, different parameters:
```powershell
./scripts/setup-workspace.ps1 `
    -WorkspaceName "AAP-RewardsLoyalty" `
    -LakehouseName "RewardsLoyaltyData" `
    -CapacityId "<production-capacity-guid>"
```

## Configuration File

`scripts/.env.fabric` is auto-generated by `setup-workspace.ps1`:

```env
FABRIC_WORKSPACE_ID=<guid>
FABRIC_WORKSPACE_NAME=AAP-RewardsLoyalty-POC
FABRIC_LAKEHOUSE_ID=<guid>
FABRIC_LAKEHOUSE_NAME=RewardsLoyaltyData
FABRIC_SQL_ENDPOINT=<server>.datawarehouse.fabric.microsoft.com
FABRIC_CAPACITY_ID=<guid>
```

> ⚠️ **Do not commit `.env.fabric` to source control.** It is listed in `.gitignore`.

## Troubleshooting

### "az: command not found"
Install Azure CLI: https://aka.ms/installazurecli

### "AADSTS700016: Application not found"
You may not have Fabric permissions. Ensure:
- Your Azure account has access to the Fabric capacity
- You've consented to the Fabric API app registration

### "Workspace creation failed (403)"
- Your account needs at least Contributor role on the Fabric capacity
- Check that the capacity ID is correct and the capacity is active

### "SQL endpoint not yet available"
The Lakehouse SQL endpoint takes 1-3 minutes to provision after creation.
Re-run `setup-workspace.ps1` to refresh the endpoint, or check in the Fabric portal.

### "Invoke-Sqlcmd not found"
Install the SqlServer module:
```powershell
Install-Module -Name SqlServer -Scope CurrentUser -Force
```
The script falls back to `System.Data.SqlClient` if SqlServer module isn't available.

### "Login failed for user"
Ensure your `az login` account has access to the Lakehouse. The SQL endpoint uses AAD token auth — the same identity that created the Lakehouse should work.

### Rate limiting (429)
The scripts handle 429 responses with automatic exponential backoff. If you see repeated rate limit errors, wait a few minutes and retry.

## Teardown / Cleanup

To remove everything created by these scripts:

```powershell
# Get your workspace ID
$envConfig = Get-Content ./scripts/.env.fabric | ConvertFrom-StringData

# Delete the workspace (this also deletes the Lakehouse and all contents)
$token = (az account get-access-token --resource "https://api.fabric.microsoft.com" | ConvertFrom-Json).accessToken
$headers = @{ Authorization = "Bearer $token" }
Invoke-RestMethod -Method DELETE `
    -Uri "https://api.fabric.microsoft.com/v1/workspaces/$($envConfig.FABRIC_WORKSPACE_ID)" `
    -Headers $headers

# Clean up local config
Remove-Item ./scripts/.env.fabric
```

> ⚠️ Deleting a workspace is **permanent** and removes all items inside it.
